### 1.1.2 🎯 Amazon KDP（キンドル・ダイレクト・パブリッシング）とは

#### 1.1.2.1 **定義**

**Amazon KDP** = Amazon が提供する **電子書籍・紙の本の自主出版プラットフォーム**

```
あなたの原稿
  ↓
KDP にアップロード
  ↓
Amazon Kindle ストアで販売
  ↓
読者が購入 → あなたが報酬獲得
```

#### 1.1.2.2 **簡単に言うと**

「自分で書いた本を、Amazon を通じて世界中に販売できるサービス」

---

### 1.1.3 💡 KDP でできる 5 つのこと

#### 1.1.3.1 1️⃣ **Kindle 電子書籍の出版**

**最も一般的な使い方**

```
EPUB / PDF / Word 形式の原稿
  ↓
KDP にアップロード
  ↓
Kindle ストア で販売開始
    - Kindle Paperwhite
    - Kindle Oasis
    - iPhone / iPad の Kindle アプリ
    - Windows / Mac の Kindle アプリ
    - ブラウザの Kindle Cloud Reader
```

**対応フォーマット:**
- EPUB（推奨）
- PDF（固定レイアウト向け）
- Microsoft Word（.doc / .docx）
- HTML

---

### 1.1.4 🗂️ 電子書籍フォーマットの基礎（EPUB / PDF / KDP 変換）

#### 1.1.4.1 EPUB
- 国際標準（W3C）が定める電子書籍フォーマット
- 再フロー型で、多種多様な端末に対応
- KDP では推奨フォーマットで、アップロード時に最適化処理がされる
- 特徴: CSS によるスタイル調整、目次（ナビゲーション）

#### 1.1.4.2 PDF
- 固定レイアウト確認や印刷レビューに向く
- 図版や紙面イメージを保ちたい技術資料で有効
- リフロー型電子書籍の本文入稿は `EPUB` を優先

#### 1.1.4.3 KDP 側の内部変換
- KDP に `EPUB` をアップロードすると、Amazon 側で Kindle 向け内部形式へ最適化される
- そのため、原稿作成段階では `EPUB` と確認用 `PDF` を用意しておけば十分
- 旧来の Kindle 専用形式は、新規ワークフローでは通常不要

> 体系的な流れ
> 1. 執筆は Word/Markdown/HTML などで行う
> 2. `EPUB` を生成して本文入稿用に使う
> 3. `PDF` を生成して固定レイアウトや印刷見えを確認する
> 4. `*-kdp-registration.md` を見ながら KDP にメタデータを転記する

---

#### 1.1.4.4 2️⃣ **Kindle Unlimited（読放題プログラム）に参加**

**追加収入を得るしくみ**

```
Kindle Unlimited（読放題）に登録
  ↓
読者が購読料（日本: 980 円/月）で利用
  ↓
あなたの書籍が読まれた時間に応じて報酬
  ↓
月間プール（約 $40 million）から配分
```

**メリット:**
- ✅ 購入者以外からも収入
- ✅ 読放題ユーザーへのリーチ
- ✅ 購入 + 読放題 = ダブル収入

**デメリット:**
- ❌ 独占契約（他のプラットフォーム販売禁止）

---

#### 1.1.4.5 3️⃣ **紙の本を出版（POD）**

**同じプラットフォームで紙の本も販売**

```
Kindle 電子書籍 + 紙の本を同時出版
  ↓
読者が紙の本を注文
  ↓
Amazon が自動で印刷・配送
  ↓
あなたが利益を獲得
```

**メリット:**
- ✅ 信頼度アップ（「紙の本も出している」）
- ✅ 図書館への納入申請が可能
- ✅ 販売チャネルの拡大

**コスト:**
- 無料（Amazon が自動印刷）
- 利益は「定価 - Amazon の印刷費」

---

#### 1.1.4.6 4️⃣ **複数冊の書籍を管理**

**1 つのダッシュボードで複数の本を管理**

```
KDP ダッシュボード
  ├── 書籍①: クリーンアーキテクチャ
  ├── 書籍②: SOLID 原則ガイド
  ├── 書籍③: デザインパターン解説
  └── 書籍④: テスト駆動開発
```

**機能:**
- ✅ 各書籍の売上・ダウンロード数を個別追跡
- ✅ プロモーション / セール実施
- ✅ 一括で複数冊を無料化

---

#### 1.1.4.7 5️⃣ **全世界で販売**

**170+ の国と地域で自動的に販売**

```
日本の読者
  ↓
米国（英訳版など）
  ↓
ヨーロッパ
  ↓
アジア太平洋地域
  ↓
すべて自動で配信・販売
```

**メリット:**
- ✅ 国際的な著者になれる
- ✅ 多言語出版も可能
- ✅ 複数の通貨で販売

---

### 1.1.5 📊 Amazon KDP の 3 つの収益モデル

#### 1.1.5.1 **モデル A：通常販売（Kindle ストア）**

**最もシンプル・最初はこれ推奨**

```
あなたが価格を設定（¥99～¥99,999）
  ↓
読者が購入
  ↓
Amazon が 30～35% 取得
  ↓
あなたが 65～70% を獲得

例：
¥1,000 で販売
  → あなたの収入 = ¥700
```

**印税率の仕組み:**
- 定価 $2.99～$9.99（約¥300～¥1,000）→ **70% 獲得**（最高）
- 定価 $0.99～$2.98 または $10.00 以上 → **35% 獲得**

**特徴:**
- ✅ 最高の印税率（他の出版社より高い）
- ✅ 価格を自由に設定
- ✅ 他のプラットフォームとの併売可

---

#### 1.1.5.2 **モデル B：Kindle Unlimited（読放題）**

**追加収入・読者層の拡大**

```
書籍を「Kindle Unlimited」に登録
  ↓
読者は月 980 円で読み放題購読（日本）
  ↓
あなたの書籍が読まれたページ数に応じて報酬
  ↓
月間プール（約 $40 million）から配分
```

**報酬計算例:**
```
月間プール: $40,000,000
全書籍の読まれたページ数: 10 億ページ

あなたの書籍: 100,000 ページが読まれた
  ↓
あなたの報酬 = $40,000,000 × (100,000 / 1,000,000,000)
           = $4,000（約 ¥440,000）
```

**条件:**
- ⚠️ **独占契約**: Kindle Unlimited に登録した本は、他のプラットフォーム（Apple Books, Google Play など）での販売禁止

**向いている書籍:**
- ✅ ストーリー性がある（読み続けられやすい）
- ✅ ページ数が多い（読まれるページ数が増える）
- ✅ 頻繁に読み直される（再利用価値がある）

---

#### 1.1.5.3 **モデル C：POD（紙の本販売）**

**信頼度アップ・多様な販売チャネル**

```
Kindle 版 + Print On Demand（紙版）を販売
  ↓
読者が紙の本を注文
  ↓
Amazon が自動で印刷・配送
  ↓
定価 - 製造原価 = あなたの利益
```

**コスト:**
- 例：定価 ¥2,000、製造原価 ¥800
- あなたの利益 = ¥1,200 / 冊

**メリット:**
- ✅ 「著者」としての信頼度 UP
- ✅ ビジネス資料としての価値 UP
- ✅ 図書館への寄贈・申請が可能

**デメリット:**
- ❌ 印刷コストがかかる（売れなければ赤字）
- ❌ 配送に 3 日～1 週間かかる

### 1.1.6 ⚖️ Amazon KDP vs その他プラットフォーム

#### 1.1.6.1 **主要プラットフォーム比較**

| 項目 | Amazon KDP | Apple Books | Google Play | 出版社経由 |
|------|-----------|-----------|-----------|----------|
| **登録料** | 無料 | 無料 | 無料 | 数万円～ |
| **ISBN** | Kindleは通常不要 / 紙は無料ISBN可 | 必須購入 | 不要 | 提供 |
| **印税率** | 70%（最高） | 70% | 70% | 10～15% |
| **販売地域** | 全世界 | 全世界 | 全世界 | 限定 |
| **出版速度** | 24 時間以内 | 1～2 週間 | 1～2 週間 | 数ヶ月 |
| **Kindle Unlimited** | ✅ あり | ❌ なし | ❌ なし | ❌ なし |
| **紙の本** | ✅ POD 可 | ✅ POD 可 | ❌ 不可 | ✅ 可能 |
| **複数プラットフォーム** | ❌ 独占禁止* | ✅ 可能 | ✅ 可能 | ✅ 可能 |

**\* Kindle Unlimited に参加した場合のみ独占契約**

### 1.1.7 ✅ Amazon KDP の メリット（9 つ）

#### 1.1.7.1 **初期費用を抑えやすい**
```
- KDP の登録：¥0
- ファイルアップロード：¥0
- 配信開始自体に初期費用はかからない
- ただし表紙制作・校正・広告などは別途発生する場合あり
```

#### 1.1.7.2 **高い印税率**
```
- 70% のロイヤリティ（業界最高水準）
- 出版社経由（10～15%）より圧倒的に高い
- あなたが大部分の売上を獲得
```

#### 1.1.7.3 **比較的早く販売開始**
```
- Amazon の確認・審査後に公開
- 通常は数時間～48 時間程度で Amazon Kindle ストアに反映
- 従来の出版より短期間で進めやすい
```

#### 1.1.7.4 **世界規模で販売**
```
- 170+ の国と地域で販売
- 言語ごとに異なる価格を設定可
- 国を跨いだ販売収入
```

#### 1.1.7.5 **ISBN の負担が少ない**
```
- Kindle 電子書籍は通常 ISBN 不要
- 紙書籍では KDP の無料 ISBN を選べる場合がある
- 独自 ISBN が必要なら別途取得も可能
```

#### 1.1.7.6 **Kindle Unlimited で追加収入**
```
- 購入者とは別の読放題ユーザーから収入
- 月間プール（$40M+）から配分
- 読まれるほど収入増加
```

#### 1.1.7.7 **いつでも修正・更新**
```
- ファイルを差し替え可能
- 説明文をいつでも変更可
- 価格をいつでも変更可
- 修正から反映まで数時間
```

#### 1.1.7.8 **詳細な売上分析**
```
- 日単位の売上追跡可
- 地域別・価格帯別の分析
- Kindle Unlimited ページ数の追跡
```

#### 1.1.7.9 **マルチフォーマット対応**
```
- Kindle（電子）
- 紙の本（POD）
- 音声本（Audible）*
- 複数形式で同時販売
```

---

### 1.1.8 ❌ Amazon KDP の デメリット（5 つ）

#### 1.1.8.1 **Kindle Unlimited は独占契約**
```
- Kindle Unlimited に登録すると独占契約に同意
- Apple Books, Google Play では販売不可
- 他のプラットフォームの自由度がなくなる
```

#### 1.1.8.2 **Amazon 専用の ISBN**
```
- KDP の ISBN は Amazon 専用
- 他のプラットフォームでは使用不可
- 複数プラットフォーム販売には別途 ISBN 購入が必要
```

#### 1.1.8.3 **競争が激しい**
```
- 毎日数千の書籍がアップロードされる
- タイトル決定が重要（検索に引っかかるか）
- マーケティングが自分で必要
```

#### 1.1.8.4 **マーケティングは自分で**
```
- Amazon は宣伝してくれない
- SNS / ブログ / メールでの自主宣伝必須
- 初期のレビュー獲得が困難
```

#### 1.1.8.5 **Amazon の方針変更**
```
- Amazon の利用規約が変わる可能性
- 価格設定ルールの変更
- プラットフォーム政策の急変
```

### 1.1.10 📝 KDP に向かない場合

#### 1.1.10.1 **こういう人は別の選択肢を検討**

- 😞 費用をかけてでも丁寧な編集・デザインが欲しい
  → **従来出版社経由** の出版
  
- 😞 複数のプラットフォーム（Apple Books, Google Play）で同時販売したい
  → **独立出版社**（IngramSpark など）を検討、または部分的に ISBN 購入
  
- 😞 Amazon 以外（例：日本のみ販売）に特化したい
  → **ローカル出版社**と契約
  
- 😞 マーケティングが嫌 / 得意でない
  → **従来出版社**に全てお任せ

### 1.2.2 🚀 アカウント作成準備チェック

実際に始める前に、以下を用意してください：

- [ ] インターネット接続
- [ ] メールアドレス（Amazon 用）
- [ ] 電話番号（本人確認用）

### 2.1.4 📝 ステップ 1: Markdown 原稿の準備

```
clean-architecture/
├── 01-introduction/
│   ├── 01-overview.md
│   ├── 02-why-clean-architecture.md
│   └── 03-key-concepts.md
├── 02-core-principles/
│   ├── 01-single-responsibility.md
│   ├── 02-open-closed.md
│   └── ...
└── ...
```

#### 2.1.4.2 **Markdown ファイルのフォーマット要件**

| 要件 | 詳細 |
|------|------|
| **文字エンコーディング** | UTF-8 |
| **改行コード** | LF (Unix 形式) |
| **ファイル名** | 01-*, 02-* (ソート順が大切) |
| **見出し** | # H1, ## H2, ### H3 を適切に使用 |
| **画像** | 相対パスで参照（.png / .jpg / .gif） |
| **リンク** | 相対パスまたは URL |

#### 2.1.4.3 **チェックリスト**

```
☐ すべての Markdown ファイルが UTF-8 で保存されている
☐ ファイル名の番号順が正しい（01-, 02-, 03-...）
☐ 見出しが階層化されている（H1 は最上位のみ）
☐ 不要な空白行や余分な改行がない
☐ 画像ファイルが正しくフォルダに配置されている
☐ 外部リンクは全て有効
```

### 2.1.9 ❌ よくあるエラーと解決方法

#### 2.1.9.1 **エラー 0: ビルドコマンドが見つからない / 実行できない**

**原因**: `Pandoc` や `Node.js` が未インストール、または初回の submodule 初期化が未実施

**対策**:
- ✅ `pandoc --version` と `node -v` が成功するか確認
- ✅ PDF 生成時は `Chrome` または `Edge` がインストールされているか確認
- ✅ 初回 clone 後は `git submodule update --init --recursive` を実行
- ✅ 実行用 wrapper（利用者向け入口）として `./.github/skills-config/ebook-build/invoke-build.ps1` を実行

#### 2.1.9.2 **エラー 1: 日本語が文字化けする**

**原因**: ファイルエンコーディングが UTF-8 でない

**解決方法**:
```powershell
# PowerShell で UTF-8 に変換
$file = Get-Content -Path "file.md" -Encoding utf8
$file | Out-File -Path "file.md" -Encoding UTF8
```

#### 2.1.9.3 **エラー 2: 画像が表示されない**

**原因**: 画像パスが相対パスでない、または支援形式でない

**確認事項**:
- ✅ 画像は PNG, JPG, GIF に対応
- ✅ 相対パスを使用（「./images/photo.png」）
- ✅ 画像サイズが適切（最大 4000x4000px）

#### 2.1.9.4 **エラー 3: メタデータが読み込まれない**

**原因**: metadata.yaml の YAML 形式が不正

**対策**:
- ✅ インデント（スペース 2 個）を統一
- ✅ シングルクォート / ダブルクォートを統一
- ✅ YAML バリデーター（online-yaml-validator.appspot.com）で確認

#### 2.1.9.5 **エラー 4: 目次が生成されない**

**原因**: Markdown の見出しが H2（##）以下のみ

**対策**:
- ✅ 最上位の見出しは H1（#）を使用
- ✅ 階層構造を正しく設定

### 2.3.3 🎯 ebook-build スキルとは

`ebook-build` は、Markdown で書かれた書籍をワンコマンドで **EPUB / PDF / KDP 登録 Markdown** に変換するための再利用可能なビルドフローです。

このリポジトリでは `.github/skills-config/ebook-build/` の **実行用 wrapper** から実行し、共有リポジトリ（shared repo）である `shared-copilot-skills/ebook-build` の実装へ委譲します。GitHub Copilot Agent モードからの利用にも、PowerShell からの直接実行にも対応しています。

> ⚠️ このスキルが自動化するのは **出版準備** までです。KDP 管理画面への最終アップロード、入力確認、公開申請は人が実施してください。

#### 2.3.3.1 できること

1. 番号付き Markdown チャプター構造（`01-`, `02-` ...）からコンテンツを収集
2. 専用の **メタデータ**・**スタイルシート** を適用して EPUB を生成
3. HTML + ローカル Chrome/Edge を使って固定レイアウト寄りの PDF を生成
4. `*.metadata.yaml` と `*.kdp.yaml` から KDP 登録用 Markdown を生成
5. 成果物を `ebook-output/` に出力

#### 2.3.3.2 スキルのファイル構成

利用側リポジトリ（consumer repo）で管理する設定と、共有リポジトリ（shared repo）の `ebook-build` 実装に分かれています。

## 3.1 メタデータと価格設定の最適化 {#section-03-advanced-topics-01-metadata-and-pricing}


### 3.1.1 📖 このセクションで学べること

このセクションを読むと、以下が理解できます：

- ✅ メタデータが売上に与える影響
- ✅ キーワード選定と KDP 検索の最適化
- ✅ 説明文（商品説明）の効果的な書き方
- ✅ カテゴリ選択による流入増加
- ✅ 価格戦略（心理価格、競争分析など）
- ✅ ロイヤルティプランの選択基準
- ✅ 売上データ分析と改善方法

**学習時間**: 約 40 分

---

### 3.1.2 🎯 メタデータが重要な理由

#### 3.1.2.1 **メタデータとは**

```
メタデータ = 書籍について説明するデータ

具体的に:
├─ タイトル・サブタイトル
├─ 著者名
├─ 説明文（書籍説明）
├─ キーワード（タグ）
├─ カテゴリ
├─ 表紙画像
└─ その他（出版社、出版日など）
```

#### 3.1.2.2 **メタデータが売上に与える影響**

```
Google での SEO と同じく、KDP でも検索順位が重要：

読者の購入プロセス:
1. Amazon Kindle で検索
   → キーワードマッチングスコア（メタデータが評価）
2. 検索結果から数冊を比較
   → タイトル、説明、評価、価格を確認（メタデータ）
3. 購入決定
   → キャッチーなタイトル＆説明が勝率UP

結論: メタデータが良い ≈ 売上が 3～5 倍
```

---

### 3.1.3 📝 ステップ 1: タイトル最適化

#### 3.1.3.1 **タイトルの役割**

```
タイトル = 最初の第一印象 = 最も重要な要素

KDP での表示順序:
1. 検索結果画面: タイトルのみ表示
2. 詳細ページ: タイトル + サブタイトル

戦略: 検索キーワードを含め、かつ読みやすく
```

#### 3.1.3.2 **Good タイトル vs Bad タイトル**

| Bad ❌ | Good ✅ | 理由 |
|--------|--------|------|
| 本 | クリーンアーキテクチャ完全ガイド | キーワード含む + 説明的 |
| アーキテクチャ | アーキテクチャ設計パターン入門 | 検索対象キーワード明確 |
| Chapter 1 | SOLID 原則 完全マスターガイド | 対象読者が理解できる |

#### 3.1.3.3 **効果的なタイトル構造**

```
【主キーワード】 + 【サブキーワード】 + 【修飾語】

例:
「クリーンアーキテクチャ」（主キーワード）
+ 「完全ガイド」（サブキーワード）
+ 「実装例とベストプラクティス」（修飾語）
= 「クリーンアーキテクチャ完全ガイド: 実装例とベストプラクティス」
```

#### 3.1.3.4 **タイトルのコツ**

```
☑ 主要キーワード（1～2 個）を含める
☑ 文字数は 60 字以内（検索結果での表示制限）
☑ サブタイトルで詳細を補足（255 字まで）
☑ スペルミスがない
☑ 大文字小文字は正しいか（特に英数字）
☑ 記号 @、#、& は避ける（検索不安定化）

❌ 避けるべき:
「【クリーンアーキテクチャ】完全ガイド～初心者～」
 （記号が多すぎる、冗長）

✅ 推奨:
「クリーンアーキテクチャ完全ガイド」
```

#### 3.1.3.5 **日本語タイトルでの SEO**

```
日本語検索での最適化:

「クリーンアーキテクチャ」で検索する読者を想定
→ Kindle での検索結果順位を高める
→ タイトルに「クリーンアーキテクチャ」を含める

読者の検索キーワード例:
- クリーンアーキテクチャ
- アーキテクチャ設計
- ソフトウェア設計
- SOLID 原則
```

### 3.1.7 💰 ステップ 5: 価格設定の最適化

#### 3.1.7.1 **価格設定の基本原理**

```
【価格 × 販売数 = 売上】

価格が高い:
  → 販売数が少なくなる傾向
  → 単価は高いが、売上が低い？

価格が安い:
  → 販売数が増える傾向
  → 単価は低いが、売上が高い？

【最適価格】 = 売上を最大化する価格点
```

#### 3.1.7.2 **価格帯別の戦略**

##### 3.1.7.2.1 **低価格戦略（¥490～¥980）**

```
メリット:
✅ 購買ハードルが低い
✅ 販売数が増える傾向
✅ 新規作者が上位ランクを狙いやすい

デメリット:
❌ ロイヤルティが少ない
❌ 35% プランしか選べない（¥490 の場合）

こんな時に推奨:
• 新作で知名度ゼロ
• 他の書籍の宣伝用
• Kindle Unlimited 参加を優先
```

##### 3.1.7.2.2 **中価格戦略（¥1,480～¥2,480）【推奨】**

```
メリット:
✅ 70% ロイヤルティが適用される
✅ 競争相手が少ない（初心者は高価格を敬遠）
✅ 技術書の標準価格

デメリット:
❌ 価格に見合う品質が必要
❌ よほど良い本でないと売れない

こんな時に推奨:
• 十分に完成度の高い本
• 著者に信頼度がある
• 専門性が高い技術書

推奨具体例:
¥1,490 【強く推奨】
  → $9.99 USD（米ドル）との最適バランス
  → 技術書としての「安くはないが、妥当」感
```

##### 3.1.7.2.3 **高価格戦略（¥2,980～¥5,000）**

```
メリット:
✅ 高額なロイヤルティ
✅ 「専門的で価値がある」というシグナル

デメリット:
❌ 販売数が激減
❌ よほど有名著者でないと売れない

こんな時に推奨:
• 業界で著名な著者
• 既にベストセラー本がある
• 内容が希少価値が高い（失敗リスク高）
```

#### 3.1.7.3 **通貨換算での価格設定**

KDP は米ドル主導の価格付けです：

```
推奨の価格対応表:

USD → JPY (日本円)
$2.99 → ¥490   (35% プラン)
$3.99 → ¥600
$4.99 → ¥750
$7.99 → ¥1,200
$9.99 → ¥1,480  【推奨】
$14.99 → ¥2,250
$19.99 → ¥3,000
```

為替変動に対応するため、Amazon は自動で円価格を調整します。

### 3.2.2 🎯 Kindle Unlimited とは

#### 3.2.2.1 **定義**

```
Kindle Unlimited （KU）= Amazon の「読放題」プログラム

読者側:
  月額 ¥980 で、200 万冊以上が読み放題

著者・出版社側:
  読者が読んだページ数に応じて報酬獲得
```

#### 3.2.2.2 **KU の全体像**

```
読者（購読者）
  ↓ 月額 ¥980 を支払い
  ↓
Amazon Kindle Unlimited プール
  ↓ 月間約 $40 million（約 600 万円）を配分
  ↓
著者・出版社（あなた）
  ↓ ページ読破数に応じて分配
  ↓
あなたのアカウント（日本）
  ↓
月末に振込（例: 月 5 万円など）
```

---

### 3.2.3 💡 Kindle Unlimited に参加するメリット・デメリット

#### 3.2.3.1 **メリット✅**

##### 3.2.3.1.1 1️⃣ **新しい収入源**

```
通常の販売：
  読者が ¥1,490 で購入
  → あなたのロイヤルティ: ¥1,043 (70%)

KU での読破:
  読者は月額 ¥980 利用
  → 月間プール $40M を全著者で分配
  → あなたが 100 ページ読まれた場合: 約 ¥100～300
```

##### 3.2.3.1.2 2️⃣ **読者リーチの拡大**

```
通常販売：
  購入をためらう層 → 購入しない

KU 参加：
  購入をためらう層 → 読放題で試し読み
  → 1000 ページ読まれたら: 約 ¥1,000～3,000
```

##### 3.2.3.1.3 3️⃣ **Kindle Unlimited 専用マーケティング機能**

```
Amazon が KU に未登録の本より KU 本を優先表示
  → 検索順位が高くなりやすい ✅
  → クリック数（ページビュー）が増える
  → その結果、販売数が増える
```

##### 3.2.3.1.4 4️⃣ **借入（バロー）に対応**

```
KU では「購入」ではなく「借入」

読者が「借りる」感覚 = 購入ハードル低い
  → 購入数が増える可能性
```

#### 3.2.3.2 **デメリット❌**

##### 3.2.3.2.1 1️⃣ **独占契約の強制**

```
KU に登録すると:
  その本を【KU 専用】にする必要があり
  ↓
  他のプラットフォームでは販売禁止：
    ❌ Apple Books（iBooks）
    ❌ Google Play Books
    ❌ Kobo
    ❌ 自社ウェブストア
    
【重要】日本以外（米国など）での販売も不可
  → 世界中の Kindle.com でのみ販売可能
```

##### 3.2.3.2.2 2️⃣ **報酬が不確定**

```
販売での報酬（確定的）：
  $9.99 で販売 → ロイヤルティ $6.99（70%）
  【確実】

KU での報酬（不確定）：
  月間プール $40M / 全著者が読んだページ数
  → 月ごとに変動（平均 $0.004～0.005/ページ）
  → 月によって大きく変わる可能性
    （プール額が減ると報酬も減少）
```

##### 3.2.3.2.3 3️⃣ **短期的には売上が減る可能性**

```
シナリオ：

【参加前】
  月 50 冊 × ¥1,043 = ¥52,150

【参加直後】
  月 200 回借用 × 500 ページ読破
  = 月間 100,000 ページ
  = ¥40,000～500,000（変動大）

参加直後は販売数が落ちることが多い
  → 読者が「購入」から「借入」に切り替え
  （ただし、上手くいけば総売上が増える）
```

### 3.2.4 📋 KU 参加の判断フロー

#### 3.2.4.1 **参加すべきか迷った場合の判断表**

| 状況 | 判断 | 理由 |
|------|------|------|
| **初出版で知名度ゼロ** | **KU 推奨** ✅ | リーチ拡大で販売数増加 |
| **既に複数冊ベストセラーがある** | **非参加推奨** | 独占契約が足かせ |
| **複数言語（日本語＋英語）での販売戦略** | **非参加推奨** | 海外での販売が制限 |
| **シリーズ化予定（第 2 巻、第 3 巻）** | **KU 推奨** | シリーズ全冊を KU で |
| **技術書（アップデート多い）** | **非参加推奨** | 更新版を別の販路で売りたい |

### 3.2.6 📊 ステップ 2: KU での報酬計算

#### 3.2.6.1 **KU での報酬の仕組み**

```
【月間プール】:
  毎月 Amazon が $40 million を割り当て
  （例: 2024 年 3 月 = $41.5M）

【配分方法】:
  プール総額 ÷ 月間全ページ読破数 = 1 ページあたりの報酬

例: 2024 年 3 月（実例）
  プール: $41,500,000
  全著者の読破ページ数: 4 billion ページ
  1 ページあたり: $0.0104
  
  あなたの本が 10,000 ページ読まれた:
  報酬 = 10,000 × $0.0104 = $104

【重要】:
  プールが減ると報酬も減少
  → 毎月変動する（当てにならない側面あり）
```

#### 3.2.6.2 **報酬の実例**

##### 3.2.6.2.1 **シナリオ: クリーンアーキテクチャ本の例**

```
書籍: クリーンアーキテクチャ完全ガイド
ページ数: 500 ページ

月 1: 100 人が 100% 読破
  = 100 × 500 = 50,000 ページ
  = 50,000 × $0.0104 = $520 ≈ ¥78,000

月 2: 150 人が 80% 読破
  = 150 × 400 = 60,000 ページ
  = 60,000 × $0.0105 = $630 ≈ ¥94,500

月 3: 50 人が 100% 読破
  = 50 × 500 = 25,000 ページ
  = 25,000 × $0.0098 = $245 ≈ ¥36,750
  （プール額が減った月）
```

#### 3.2.6.3 **KU 報酬の確認方法**

```
KDP ダッシュボール → 「レポート」
  ↓
「KU レポート」をクリック
  ↓
以下が表示:
  ・ページビュー数（日別）
  ・読破ページ数（日別）
  ・推定報酬（日別）
  ・月間合計
```

### 3.2.7 🎯 KU vs 単独販売の比較

#### 3.2.7.1 **収入モデルの比較**

##### 3.2.7.1.1 **シナリオ: 月 100 冊売れる本の場合**

**【単独販売】（KU 非参加）**

```
販売数: 100 冊/月
定価: ¥1,490
ロイヤルティ: 70% = ¥1,043/冊

月間売上: 100 × ¥1,043 = ¥104,300

特徴:
✅ 収入が確定的
✅ 他販路での販売可能
❌ KU 読者にリーチできない
```

**【KU 参加】（参加戦略）**

```
販売数: 30 冊/月
KU・読破: 10,000 ページ/月

販売売上: 30 × ¥1,043 = ¥31,290
KU 報酬: 10,000 × ¥9.5 = ¥95,000
（1 ページ平均 ¥0.0095 と仮定）

月間売上: ¥31,290 + ¥95,000 = ¥126,290

特徴:
✅ 総売上が増加（単独販売比 121%）
✅ KU 専用マーケティング機能の恩恵
❌ 独占契約で他販路にバラれない
❌ 報酬が月ごとに変動
```

#### 3.2.7.2 **どちらを選ぶべきか?**

```
✅ KU 参加すべき場合:
  • 初出版で知名度がない
  • 販売数 50 冊/月 以下
  • 日本マーケット中心
  • シリーズ化予定

❌ 単独販売すべき場合:
  • 既に知名度がある著者
  • 月間販売数 100 冊以上
  • 国際販売を重視
  • 技術書で頻繁に更新
```

### 3.2.8 📈 ステップ 3: KU での成功戦略

#### 3.2.8.1 **KU に適した書籍の特性**

### 3.2.12 ❓ KU よくある質問

#### 3.2.12.1 **Q1: KU に登録しても販売数は減らない？**

A: 基本的には減ります。理由：
```
読者が「購入」から「借入」に切り替え
  → 購入冊数が 30～70% 減少
  → ただし、KU による報酬がカバーできれば OK
```

#### 3.2.12.2 **Q2: 一度 KU に登録したら、やめられない？**

A: やめられます。ただし条件があります：
```
90 日の契約終了時点で「更新をスキップ」
  → 自動的に KU から除外
  → 他販路での販売が可能に

ただし一度 KU をやめると、再登録は 90 日後以降
```

#### 3.2.12.3 **Q3: KU に登録した本を他の言語（英語版など）で出版できる？**

A: はい、別タイトルなら OK：
```
日本語版: 「クリーンアーキテクチャ完全ガイド」(KU 登録)
英語版: 「Clean Architecture Complete Guide」(別タイトル、KU なし)
```

#### 3.2.12.4 **Q4: KU でも価格設定できる？**

A はい。ただしルールあり：
```
価格: $2.99～$9.99 の範囲で設定可能
注意: KU では販売数が減るため、価格を下げるのが定石
```
