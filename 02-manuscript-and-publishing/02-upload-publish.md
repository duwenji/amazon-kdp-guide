# 02. KDP へのアップロードと出版

## 📖 このセクションで学べること

このセクションを読むと、以下が理解できます：

- ✅ KDP のダッシュボード操作方法
- ✅ タイトル（本）を新規作成する手順
- ✅ AZW3 ファイルをアップロードする方法
- ✅ 表紙画像の設定
- ✅ 出版前チェック（品質チェック機能）
- ✅ 出版ボタンを押す前の最終確認
- ✅ 出版審査プロセス
- ✅ 出版完了後の確認方法

**学習時間**: 約 25 分

---

## 🎯 アップロード～出版の全体フロー

```
① KDP ダッシュボードにログイン
  ↓
② 「新規タイトルを作成」をクリック
  ↓
③ 書籍情報（タイトル、著者など）を入力
  ↓
④ AZW3 ファイルをアップロード
  ↓
⑤ 表紙画像をアップロード
  ↓
⑥ 価格・ロイヤルティプランを設定
  ↓
⑦ 出版前チェック（品質チェック）を実行
  ↓
⑧ 「出版」ボタンをクリック
  ↓
⑨ Amazon による審査（24～48 時間）
  ↓
✅ Kindle ストアで販売開始
```

---

## 🚀 ステップ 1: KDP ダッシュボードへログイン

### **URL**

```
https://kdp.amazon.com
```

### **ログイン手順**

1. 上記 URL を開く
2. Amazon アカウントでサインイン
3. 出版者情報がまだ登録されていない場合：
   - 「出版を開始する」→「Kindle eBooks」を選択
   - 出版者情報（ペンネーム、メールアドレス）を入力
   - 利用規約に同意

### **初回設定（単一回のみ）**

```
出版者情報の設定：
├─ 法人 vs 個人 を選択
├─ ペンネーム（著者名として使用される）
├─ 出版社名（オプション）
├─ 銀行振込口座情報（報酬受け取り用）
└─ 税務情報（W-8BEN フォーム）
```

---

## 📝 ステップ 2: 新規タイトルを作成

### **「新規タイトルを作成」ボタン**

ダッシュボード上の「新規タイトルを作成」をクリックします。

### **書籍情報の入力**

| 項目 | 説明 | 例 |
|------|------|-----|
| **言語** | 書籍の言語 | 日本語（Japanese） |
| **タイトル** | 書籍のタイトル（必須） | クリーンアーキテクチャ完全ガイド |
| **サブタイトル** | タイトルの補説（オプション） | 実践的な設計パターン |
| **著者名** | 著者名（必須） | 山田太郎 |
| **説明** | 書籍の説明（最大 4000 字） | 50～200 字推奨 |
| **シリーズ名** | シリーズものの場合（オプション） | 開発者ガイドシリーズ |
| **出版社** | 出版社名（オプション） | 個人出版の場合は空白 |

### **説明文のコツ**

良い説明文：
```
長さ: 50～200 字（最初の 100 字が KDP ストア表示）
内容: 何を学べるか、対象者は誰か
例:
「クリーンアーキテクチャの基本から応用まで、
実装例とベストプラクティスを詳しく解説した
完全ガイド。SOLID 原則、デザインパターン、
レイアーの分離方法を学べます。」
```

## 📤 ステップ 3: 原稿ファイル（AZW3）をアップロード

### **ファイルアップロード方法**

1. 「本の内容」セクションで「アップロード中…」をクリック
2. ファイル選択ダイアログで `clean-architecture.azw3` を選択
3. アップロード完了を待つ（ファイルサイズによって数分）

### **アップロード可能なフォーマット**

| フォーマット | 推奨度 | 説明 |
|-------------|--------|------|
| **AZW3** | ⭐⭐⭐⭐⭐ | Kindle Format 8（推奨） |
| **EPUB** | ⭐⭐⭐⭐ | 国際標準フォーマット |
| **PDF** | ⭐⭐⭐ | 固定レイアウト向け |
| **.docx** | ⭐⭐ | Word ドキュメント |
| **HTML** | ⭐⭐⭐⭐ | HTML5 対応 |

### **アップロード時の確認**

```
☐ ファイルサイズが 50MB 以下である
☐ ファイル形式が正しい（.azw3）
☐ ファイル名に日本語が含まない
☐ アップロードエラーが表示されていない
```

### **よくある AZW3 エラーと対策**

| エラー | 原因 | 対策 |
|--------|------|------|
| ファイルが読み込めない | 形式が不正 | convert-to-kindle.ps1 で再生成 |
| 目次が見つからない | EPUB の nav.xhtml が不正 | メタデータ・CSS を確認 |
| 画像が表示されない | 画像フォーマット不正 | PNG / JPG に変換 |
| 文字化けがある | エンコーディング不正 | UTF-8 で再変換 |

## 🖼️ ステップ 4: 表紙画像の設定

### **表紙画像の仕様**

| 項目 | 要件 | 説明 |
|------|------|------|
| **形式** | JPG / PNG | PNG 推奨 |
| **解像度** | 最小: 625x1000px | 3:2 のアスペクト比 |
| **推奨** | 1400x2100px | 高品質推奨 |
| **最大** | 10000x10000px | 3MB 以下 |
| **背景** | 無地 | 背景透過NG |

### **表紙のアップロード**

1. 「本の表紙」セクションで「画像をアップロード」をクリック
2. `cover.png` を選択
3. Amazon のプレビューで表示確認

### **表紙デザインのコツ**

```
KDP 表紙のポイント：
✅ タイトルがはっきり見える
✅ サムネイル表示でも読める（小さくても可読）
✅ 背景色はシンプル
✅ カテゴリに合ったデザイン
✅ フォントは見やすいもの

デザイン参考：
- Amazon Kindle 公式ガイド
- Canva（テンプレート付きデザインツール）
- GIMP / Photoshop
```

## 💰 ステップ 5: 価格とロイヤルティを設定

### **出版地域の選択**

```
☑ 全世界 （推奨）
  ├─ 日本
  ├─ 米国
  ├─ 英国
  ├─ 欧州
  └─ その他 100+ 地域
```

### **ロイヤルティプラン**

#### **プラン A: 35% ロイヤルティ**

```
対象: $2.99 以下の価格設定

例:
書籍価格: $2.99
ロイヤルティ: $2.99 × 35% = $1.05
Amazon 手数料: $0.99

あなたの収入: 約 $0.06
（推奨されません）
```

#### **プラン B: 70% ロイヤルティ（お勧め）**

```
対象: $2.99～$9.99 の価格設定

例:
書籍価格: $9.99
ロイヤルティ: $9.99 × 70% = $6.99
Amazon 手数料: $0.99

あなたの収入: 約 $6.00
（推奨です）
```

#### **日本の価格設定（JPY）**

KDP は外国（主に USD）での販売が多いため、日本国内では次のように設定します：

```
米国価格: $9.99 に連動させるのが一般的

円換算の目安:
$2.99 ≈ ¥450
$4.99 ≈ ¥750
$9.99 ≈ ¥1,500
```

### **推奨価格帯**

```
技術書（日本向け）: ¥980～¥2,980
一般向け（日本）: ¥490～¥1,980
国際向け（USD）: $4.99～$9.99
```

## ✅ ステップ 6: 出版前チェック（品質チェック）

### **自動品質チェック機能**

KDP アップロード後、Amazon は自動で以下をチェック：

```
💾 ファイル整合性チェック
  ☑ ファイル形式が有効
  ☑ メタデータが正しい
  ☑ 画像が埋め込まれている

📄 コンテンツチェック
  ☑ 目次が正しく生成されている
  ☑ ページブレークが正常
  ☑ テキストエンコーディングが正しい

👥 コンテンツポリシーチェック
  ☑ 著作権侵害がない
  ☑ 個人情報が適切
  ☑ 暴力・違法コンテンツがない
```

### **プレビュー確認（推奨）**

```
1. 「プレビューを見る」をクリック
2. 複数デバイスで確認：
   ☑ Kindle Paperwhite (6inch)
   ☑ Kindle Oasis (7inch)
   ☑ Kindle Fire HD
   ☑ iPhone アプリ
   ☑ iPad アプリ
3. 各ページをめくって問題ないか確認
```

### **よくある警告メッセージ**

| メッセージ | 意味 | 対応 |
|-----------|------|------|
| 「表紙画像が小さい」 | 最小解像度以下 | 1400x2100px 以上に修正 |
| 「メタデータが不完全」 | 説明文がない | 説明文を 50 字以上に |
| 「ISBN が見つからない」 | ISBN 未登録 | Kindle のみの場合は不要 |

## 🎉 ステップ 7: 出版ボタンをクリック

### **最終確認チェックリスト**

出版ボタンの前に、このリストを全て確認：

```
メタデータ:
☑ タイトルが正確
☑ 著者名が正確
☑ 説明が読みやすい
☑ 言語が日本語に設定

ファイル:
☑ AZW3 ファイルが正しくアップロード
☑ 品質チェックエラーがない
☑ プレビューで表示が正常

表紙:
☑ 表紙画像がアップロード
☑ 解像度が要件を満たしている
☑ タイトルが見える

価格:
☑ 価格が妥当
☑ ロイヤルティプランが選択済み
☑ 対象地域が設定済み

法的:
☑ 著作権侵害がない
☑ 利用規約に同意している
☑ 出版者情報が登録済み
```

### **「出版」ボタンをクリック**

すべての確認が完了したら、「出版」ボタンをクリック：

```
出版ボタンをクリック
  ↓
「出版を確認しますか？」ダイアログが表示
  ↓
「はい、出版します」をクリック
  ↓
✅ 出版処理開始
```

## 📊 ステップ 8: 出版審査と監視

### **Amazon による審査プロセス**

```
タイムライン:

出版ボタン
  ↓
（注文処理中） 30 分～1 時間
  ↓
（審査中） 12～24 時間
  ↓
✅ 販売中（Kindle ストアに表示）
```

### **審査内容**

Amazon の自動審査で以下をチェック：

```
🔍 コンテンツのチェック
  ✓ 不適切な内容がないか
  ✓ 著作権侵害がないか
  ✓ メタデータが正確か

📦 フォーマットのチェック
  ✓ ファイルが正しく変換されているか
  ✓ 目次が自動生成されているか
  ✓ 画像が正しく表示されるか
```

### **審査結果の確認**

ダッシュボード「個別に本を管理」セクションで確認：

```
【状態表示】

審査中: 「注文中」または「承認待ち」
✅ 販売中: 「公開中」
❌ 却下: 「却下されました」
  → メイルで却下理由の連絡あり
```

### **却下された場合の対応**

Email で却下理由が届きます：

```
よくある却下理由:
1. メタデータが不適切
2. 表紙画像の品質が低い
3. 著作権侵害の疑い
4. ファイル形式エラー

対応方法:
① 理由を確認
② ファイルを修正
③ 修正版を再アップロード
④ 再度出版申請
```

## 🌍 ステップ 9: Kindle ストアで販売開始

### **販売開始の確認**

```
1. ダッシュボードで「公開中」を確認
2. Amazon Kindle ストアで検索して確認
3. Kindle アプリで購入可能か確認
```

### **Kindle ストアでの表示例**

```
Amazon 検索結果画面:
┌─────────────────────────────┐
│  [表紙画像]                   │
│  クリーンアーキテク チャ     │
│  完全ガイド                  │
│  著者: 山田太郎              │
│  価格: ¥1,500 / $9.99         │
│  評価: ★★★★☆ (5件)          │
│  「今すぐ購入」ボタン         │
└─────────────────────────────┘
```

### **初期段階でできること**

```
✅ 常に可能
  ・ 価格の変更
  ・ 説明文の更新
  ・ キーワード（タグ）の追加
  ・ カテゴリの変更

⏳ 24 時間後に可能
  ・ カバー画像の更新
  ・ AZW3 ファイルの更新
  ・ タイトル名の変更（要注意：Kindle 名が変わる）
```

## 🎯 出版後のマーケティング

販売を促進するためのアクション：

```
短期（出版直後～1 週間）:
1. SNS で告知（Twitter / Facebook など）
2. ブログ記事で告知
3. メールリストに通知

中期（1 週間～1 ヶ月）:
1. Amazon の宣伝ツールを使用
2. キーワード広告（Amazon Ads）
3. インフルエンサーへの無料配布

長期（1 ヶ月以降）:
1. レビューを増やす施策
2. カテゴリ内での順位改善
3. 次の書籍の出版検討
```
