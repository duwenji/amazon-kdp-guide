# Contributing

このリポジトリへの改善提案や修正は歓迎します。

## 基本方針
- 初学者でも再現しやすい説明を優先してください。
- 出版実務で役立つチェックポイントや注意点を明記してください。
- 大きな構成変更は `Issue` で背景共有してから進めてください。

## 推奨フロー
1. `Issue` で変更理由を共有する
2. branch を作成する
3. `docs/` 配下を更新する
4. `./.github/skills-config/ebook-build/invoke-build.ps1` を実行する
5. `VALIDATION_CHECKLIST.md` を確認する
6. `Pull Request` を作成する

## レビュー観点
- リンク切れがないか
- 手順が再現可能か
- Amazon KDP の説明に誤解を招く表現がないか
- 公開前チェックの観点が十分か
